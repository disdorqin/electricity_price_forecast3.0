
关键设计模式：
1. **子进程隔离**：每个模型通过 `subprocess.run([python, "-m", "runners.run_{model}", json_params])` 调用
2. **配置分层**：YAML 默认值 → CLI 参数覆盖 → dataclass 校验
3. **统一输出格式**：所有 runner 输出 `["时刻", "预测值"]` 列的 CSV

## 二、目标架构

### 2.1 标准模型目录结构

每个模型统一为以下结构（仿照 SOTA 模型格式）：

```
{ModelName}/
├── __init__.py
├── dataprocess.py     # 数据加载、特征工程、cutoff 处理
├── model.py           # 模型定义（神经网络/树模型/调用外部库）
├── pipeline.py        # 训练 + 推理编排（对外的唯一接口）
├── config.py          # 配置 dataclass（可选，简单模型可内联）
├── requirements.txt   # 模型专属依赖
└── README.md          # 使用说明
```

**接口约定（pipeline.py 必须暴露的方法）**：

```python
class ModelPipeline:
    def train(self, data_path: str, target: str, train_start: str, train_end: str, **kwargs) -> dict:
        """训练模型，返回训练指标 dict"""
        ...

    def predict(self, data_path: str, target: str, forecast_date: str, **kwargs) -> pd.DataFrame:
        """单日预测，返回 DataFrame[时刻, 预测值]"""
        ...

    def predict_range(self, data_path: str, target: str, start_date: str, end_date: str, **kwargs) -> pd.DataFrame:
        """区间预测，返回 DataFrame[时刻, 预测值]"""
        ...
```

### 2.2 统一运行入口

在项目根目录创建 `main.py`，支持三种指令：

```bash
# 日前预测 (Day-Ahead)
python main.py --pipeline predict --target dayahead --models TimeMixer,RT916,LightGBM --date 2026-06-01

# 实时预测 (Real-Time)
python main.py --pipeline predict --target realtime --models SGDFNet,TimeMixer,LightGBM --date 2026-06-01

# 综合预测 (Both)
python main.py --pipeline predict --target both --models all --date 2026-06-01

# 训练
python main.py --pipeline train --target both --models all --end_date 2026-06-01

# 评估
python main.py --pipeline evaluate --pred_path outputs/2026-06-01/ --actual_path data/actual.csv

# 融合
python main.py --pipeline fusion --target both --date 2026-06-01
```

### 2.3 并行执行策略

```
main.py
  └── predict_pipeline
       └── predict_service
            ├── ProcessPoolExecutor(max_workers=4)
            │   ├── submit(run_timemixer)    → GPU (if available)
            │   ├── submit(run_sgdfnet)      → CPU
            │   ├── submit(run_lgbm)         → CPU
            │   └── submit(run_rt916)        → GPU (if available)
            └── wait for all → merge predictions → save
```

- **CPU 模型**：LightGBM、SGDFNet（HistGradientBoosting）→ 强制 CPU
- **GPU 模型**：TimeMixer（PyTorch）、TimesFM（PyTorch/JAX）→ 可配 GPU
- **RT916**：PyTorch 模型，可用 GPU
- 使用 `concurrent.futures.ProcessPoolExecutor` 替代当前的串行 `subprocess.run`
- 每个子进程独立加载模型，避免 GIL 竞争和 GPU OOM 级联

### 4.3 并行调度实现

```python
# services/predict_service.py
from concurrent.futures import ProcessPoolExecutor, as_completed

GPU_MODELS = {"TimeMixer", "TimesFM", "RT916"}

def predict_parallel(models, params, gpu_models=None):
    gpu_models = gpu_models or GPU_MODELS
    results = {}

    with ProcessPoolExecutor(max_workers=len(models)) as executor:
        futures = {}
        for model_name in models:
            runner_module = f"runners.run_{model_name.lower()}"
            env_override = {}
            if model_name not in gpu_models:
                env_override["CUDA_VISIBLE_DEVICES"] = ""  # 强制 CPU

            future = executor.submit(
                _run_model_subprocess, runner_module, params, env_override
            )
            futures[future] = model_name

        for future in as_completed(futures):
            model_name = futures[future]
            try:
                results[model_name] = future.result()
            except Exception as e:
                logger.error(f"{model_name} failed: {e}")
                results[model_name] = None

    return results
```

### 4.4 Runner 标准接口

每个 runner 模块（`runners/run_{model}.py`）必须遵循：

```python
import sys, json
from {ModelName}.pipeline import ModelPipeline

def main():
    params = json.loads(sys.argv[1])
    pipeline = ModelPipeline()

    if params.get("action") == "train":
        result = pipeline.train(**params["train_args"])
    elif params.get("action") == "predict":
        result = pipeline.predict(**params["predict_args"])
    elif params.get("action") == "predict_range":
        result = pipeline.predict_range(**params["predict_args"])

    # 统一输出格式
    result.to_csv(params["output_path"], index=False, encoding="utf-8-sig")
    print(json.dumps({"status": "ok", "output": params["output_path"]}))

if __name__ == "__main__":
    main()
```

## 六、验证标准

每个模型重构完成后，必须通过以下验证：

1. **功能验证**：重构后的模型在至少 1 个代表月上的结果与重构前差异 < 0.1%（证明重构没有改变行为）
2. **接口验证**：通过 `runners/run_{model}.py` 子进程调用成功，输出标准 CSV 格式
3. **泄漏验证**：对修复后的模型，确认 delta_lag_1 等特征不再包含当日 RT 数据
4. **统一入口验证**：`python main.py --pipeline predict --target both --models all --date 2026-05-01` 能够并行运行所有模型并输出融合结果
