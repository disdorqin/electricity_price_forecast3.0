## 四、需要修复的问题

### 必修（阻断）

**Bug 1：TimeMixer output-dir 相对路径问题**

文件：`fusion/runners/run_timemixer_enhanced_export.py`

问题：passthrough 参数中的 `--output-dir` 是相对路径，但 subprocess 的 `cwd=TimeMixer/`，导致文件写到了 `TimeMixer/fusion_runs/...`。

修复方案：在 runner 中将 `--output-dir` 转为绝对路径：
```python
# 在构建 subprocess command 时
output_dir = getattr(args, 'output_dir', None)
if output_dir and not Path(output_dir).is_absolute():
    output_dir = str((PROJECT_ROOT / output_dir).resolve())
command.extend(["--output-dir", output_dir])
```

或者在 `pipeline_common.py` 中构建 command 时就用绝对路径（当前已经是 `str(simulation_dir)` 但可能不是绝对路径）。

**同时需要清理：**
- 删除误写到 `TimeMixer/fusion_runs/` 的目录
- 检查 `--save-checkpoint` 参数是否也有同样问题

### 建议修复（不阻断但应修）

**Bug 2：executor 串行路径 None 泄漏**

文件：`runners/executor.py` 第 37-39 行

```python
# 当前代码（串行路径）
results.append(_run_task(task))

# 应改为
result = _run_task(task)
if result is not None:
    results.append(result)
```

**Bug 3：conda-env 空字符串传递**

文件：`services/fusion_service.py` 第 47-48 行

```python
# 当前代码
if conda_env:
    cmd.extend(["--conda-env", str(conda_env)])
else:
    cmd.extend(["--conda-env", ""])  # 传空字符串，下游可能出问题

# 建议改为
if conda_env:
    cmd.extend(["--conda-env", str(conda_env)])
# 不传 --conda-env 参数，让下游使用默认行为
```

**Bug 4：TimeMixer DA 分段指标全 NaN**

TimeMixer DA 训练的 per-period 指标（1_8, 9_16, 17_24）全部显示 n=0, NaN，但 overall 有值（n=5184）。这说明分段指标统计逻辑有 bug，不影响预测结果但影响诊断。
