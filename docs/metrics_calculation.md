# 指标计算说明

本文档用于统一模型评估指标与业务指标的计算口径，避免训练、验证、排行榜和业务复盘时出现口径不一致。

## 1. 符号约定

| 符号 | 含义 |
| --- | --- |
| `n` | 参与计算的样本点数量 |
| `y_i` | 第 `i` 个样本的真实值 |
| `ŷ_i` | 第 `i` 个样本的预测值 |
| `P_da_i` | 第 `i` 个时刻的实际日前价格 |
| `P_rt_i` | 第 `i` 个时刻的实际实时价格 |
| `P̂_da_i` | 第 `i` 个时刻的预测日前价格 |
| `P̂_rt_i` | 第 `i` 个时刻的预测实时价格 |
| `q_i` | 第 `i` 个时刻的售电量，触发交易时取 `1`，否则取 `0` |

如无特殊说明，所有指标均在同一批有效样本上计算。

## 2. 常规模型指标

### 2.1 MAE

MAE 用于衡量预测值与真实值的平均绝对误差。

```text
MAE = (1 / n) * Σ |ŷ_i - y_i|
```

### 2.2 MSE

MSE 用于衡量预测值与真实值的平均平方误差，对大误差更敏感。

```text
MSE = (1 / n) * Σ (ŷ_i - y_i)^2
```

### 2.3 MAPE

MAPE 用于衡量相对误差百分比。

```text
MAPE = (100% / n) * Σ |(ŷ_i - y_i) / y_i|
```

注意事项：

- 当 `y_i = 0` 时，MAPE 分母为 0，不能直接计算。
- 实现时需要提前约定处理方式，例如剔除该点、使用极小值 `epsilon` 替代分母，或改用 SMAPE。
- 如果用于电价场景，建议优先报告 SMAPE 或带业务裁剪规则的 accuracy。

### 2.4 R2

R2 用于衡量模型解释真实值波动的能力。

```text
R2 = 1 - Σ(y_i - ŷ_i)^2 / Σ(y_i - mean(y))^2
```

其中：

```text
mean(y) = (1 / n) * Σ y_i
```

注意事项：

- `R2 = 1` 表示完全预测准确。
- `R2 = 0` 表示效果等同于始终预测真实均值。
- `R2 < 0` 表示效果差于始终预测真实均值。

## 3. 业务指标

### 3.1 SMAPE 与日前电价预测准确率

SMAPE 用于衡量日前电价预测的对称百分比误差。

为了避免电价过小对准确率造成超额影响，当某一时刻的真实日前价格或预测日前价格小于 `50` 时，计算 SMAPE 前均按 `50` 处理。

裁剪后的价格定义为：

```text
y_i_clip = max(y_i, 50)
ŷ_i_clip = max(ŷ_i, 50)
```

SMAPE 计算公式为：

```text
SMAPE = (100% / n) * Σ |ŷ_i_clip - y_i_clip| / ((|ŷ_i_clip| + |y_i_clip|) / 2)
```

日前电价预测准确率定义为：

```text
accuracy = 1 - SMAPE
```

注意事项：

- 如果 SMAPE 以百分比数值表示，例如 `12.5%`，则 accuracy 为 `87.5%`。
- 如果代码中 SMAPE 使用小数表示，例如 `0.125`，则 accuracy 为 `0.875`。
- 文档、报表和代码需要统一百分比或小数口径，避免重复乘以 `100%`。

### 3.2 SCR 价差方向准确率

SCR 用于评估模型是否正确判断价差方向。现货市场中，价差方向会影响套利盈亏。

价差定义可根据业务任务指定，例如：

```text
真实价差 y_i = P_rt_i - P_da_i
预测价差 ŷ_i = P̂_rt_i - P̂_da_i
```

符号函数定义为：

```text
sgn(x) =  1, x > 0
sgn(x) =  0, x = 0
sgn(x) = -1, x < 0
```

方向是否一致的指示函数定义为：

```text
I(x, y) = 1, x = y
I(x, y) = 0, x != y
```

SCR 计算公式为：

```text
SCR = (1 / n) * Σ I(sgn(y_i), sgn(ŷ_i))
```

也就是统计真实价差和预测价差符号一致的样本占比。

### 3.3 度电套利

度电套利用于评估基于预测日前价格进行售电决策后的单位电量套利收益。

基础版本交易规则：

```text
如果 P̂_da_i > P_da_i，则 q_i = 1
否则 q_i = 0
```

即：当预测日前价格大于实际日前价格时，认为该时刻售电量为 `1`。

该时刻售利按实际实时价格与实际日前价格的价差计算：

```text
profit_i = q_i * (P_rt_i - P_da_i)
```

总售利为：

```text
total_profit = Σ profit_i
```

总售电量为：

```text
total_volume = Σ q_i
```

度电套利为：

```text
unit_arbitrage = total_profit / total_volume
```

注意事项：

- 若 `total_volume = 0`，说明没有触发任何售电决策，度电套利不能直接计算。
- 实现时建议返回 `NaN`、`0` 或单独标记“无交易”，具体口径需要在报表中明确。
- 单位通常与输入价格单位一致，例如价格为 `元/兆瓦时` 时，度电套利也是 `元/兆瓦时`。

### 3.4 度电套利改良版

度电套利改良版在基础版本上加入预测实时价格约束，只有当模型同时判断预测实时价格高于预测日前价格，并且预测日前价格高于实际日前价格时，才触发售电。

改良版交易规则：

```text
如果 P̂_rt_i > P̂_da_i 且 P̂_da_i > P_da_i，则 q_i = 1
否则 q_i = 0
```

该时刻售利仍按实际实时价格与实际日前价格的价差计算：

```text
profit_i = q_i * (P_rt_i - P_da_i)
```

总售利、总售电量和度电套利的计算方式不变：

```text
total_profit = Σ profit_i
total_volume = Σ q_i
unit_arbitrage_improved = total_profit / total_volume
```

注意事项：

- 改良版只改变是否售电的触发条件。
- 售利计算仍使用实际实时价格减实际日前价格，即 `P_rt_i - P_da_i`。
- 与基础版本一样，若 `total_volume = 0`，需要单独处理无交易情况。

## 4. 推荐输出字段

建议评估脚本或实验结果中统一输出以下字段：

| 字段 | 含义 |
| --- | --- |
| `mae` | 平均绝对误差 |
| `mse` | 平均平方误差 |
| `mape` | 平均绝对百分比误差 |
| `r2` | R2 决定系数 |
| `smape` | 裁剪规则后的 SMAPE |
| `day_ahead_accuracy` | 日前电价预测准确率，等于 `1 - smape` |
| `scr` | 价差方向准确率 |
| `arbitrage_total_profit` | 基础版总售利 |
| `arbitrage_total_volume` | 基础版总售电量 |
| `arbitrage_unit_profit` | 基础版度电套利 |
| `arbitrage_improved_total_profit` | 改良版总售利 |
| `arbitrage_improved_total_volume` | 改良版总售电量 |
| `arbitrage_improved_unit_profit` | 改良版度电套利 |

## 5. 实现校验要点

- SMAPE 计算前先对真实日前价格和预测日前价格做 `max(value, 50)` 裁剪。
- `accuracy = 1 - SMAPE`，需要确认 SMAPE 在代码中是小数还是百分比。
- SCR 只比较价差方向，不比较价差大小。
- 度电套利的交易触发条件使用预测与实际日前价格判断，但售利使用实际实时价格与实际日前价格计算。
- 度电套利改良版只改变交易触发条件，不改变售利公式。
- 所有业务指标都需要明确样本范围，例如按公司、按日期区间、按小时或按完整评估集计算。
