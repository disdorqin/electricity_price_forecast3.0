# 融合链路 TimesFM 阻断修复与跑通计划（2026-06-17）

## 当前状态

TimeMixer 路径 bug：✅ 已修复，输出正确写入 `fusion_runs/`
executor None 过滤：✅ 已修复
conda-env 空字符串：✅ 已修复

**唯一阻断：TimesFM 在融合链路中仍然硬崩溃。**

## 根因分析

TimesFM 有两条独立的代码路径，优雅跳过只在 `predict` 链路生效：

| 链路 | 文件 | TimesFM 缺失时行为 |
|------|------|-------------------|
| predict（`main.py --pipeline predict`） | `TimesFM/pipeline.py` | ✅ log warning + return None |
| fusion（`main.py --pipeline fusion`） | `fusion/runners/run_timesfm_export.py` | ❌ raise RuntimeError → 进程退出码 1 |

fusion 链路的失败链条：
```
run_timesfm_export.py: raise RuntimeError("cached export not found")
  → subprocess 退出码 1
  → pipeline_common.py _stream_subprocess: raise CalledProcessError
  → _run_specs_with_scheduler: gpu_error → raise → 整个 task 崩溃
  → run_full_fusion_suite.py: DA 崩溃 → RT 永远不跑 → joint report 不产出
```

关键差异：`pipeline_common.py` 有 `--disable-lightgbm` 开关，但**没有 `--disable-timesfm`**。而且 `_run_specs_with_scheduler` 对单个模型失败没有 try/except，一个模型挂掉整个流水线崩溃。

## 修复方案

### 方案 A：快速修复（先跑通测试）

用 TimesFM 缓存覆盖的日期范围跑烟测。TimesFM DA/RT 缓存覆盖 2026-05-01 ~ 2026-05-10，ExtremPriceClf 数据覆盖到 2026-05-26。

**共同可测窗口：2026-05-05 ~ 2026-05-07**（3 天，两边都覆盖）

### 方案 B：根本修复（让融合链路容错）

#### B1：`fusion/runners/run_timesfm_export.py` — 缓存未命中时优雅退出

```python
# 替换 raise RuntimeError
if not _try_cached_export(...):
    logger.warning("TimesFM: no cached predictions for window %s ~ %s, skipping", start, end)
    # 写一个空 CSV（只有列头），让下游 adapter 能正常加载但贡献 0 行
    empty_df = pd.DataFrame(columns=["时刻", "预测值", "真实值"])
    empty_df.to_csv(output_path, index=False)
    sys.exit(0)  # 正常退出，不报错
```

#### B2：`fusion/pipeline_common.py` — 添加 `--disable-timesfm` 开关

在 `build_model_specs()` 中，类似已有的 `--disable-lightgbm`：
```python
if getattr(args, 'disable_timesfm', False):
    pool = [m for m in pool if m != "timesfm"]
```

在 `cli/parser.py` 中添加参数：
```python
parser.add_argument("--disable-timesfm", action="store_true", default=False)
```

#### B3：`fusion/pipeline_common.py` — 单模型失败不崩溃

在 `_run_specs_with_scheduler` 中，为每个 spec 加 try/except：
```python
for spec in gpu_specs:
    try:
        run_one(spec)
    except Exception as e:
        logger.warning(f"Model {spec.model_name} failed: {e}, skipping")
        continue
```

#### B4：`run_fixed_window_fusion.py` — adapter 加载容错

在 `_load_normalized_predictions` 中：
```python
for artifact in artifacts:
    try:
        adapter = adapter_cls(str(artifact.source), **artifact.adapter_kwargs)
        df = adapter.load().copy()
        df["model_name"] = artifact.model_name
        frames.append(df)
    except (FileNotFoundError, pd.errors.EmptyDataError) as e:
        logger.warning(f"Skipping {artifact.model_name}: {e}")
        continue
```
