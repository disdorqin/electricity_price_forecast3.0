## 三、烟测阻断问题：编码损坏（Mojibake）

### 根因

`RT916_SpikeFusionNet/pipeline.py` 的 `TARGET_MAP` 存在 **UTF-8 编码损坏**：

```python
# pipeline.py 中的损坏值（mojibake）
TARGET_MAP = {
    "dayahead": "鏃ュ墠鐢典环",    # 应该是 "日前电价"
    "realtime": "瀹炴椂鐢典环",     # 应该是 "实时电价"
}
```

### 影响

`core.py` 内部用 `"日前电价"` 做分支判断（至少 15 处），如：
- `if target == "日前电价":` —— 选择 DA 特征集
- `if CONFIG["OUTPUT"] == "日前电价":` —— 控制数据集构建
- `if cfg["OUTPUT"] == "日前电价":` —— 选择 DA backbone 模型

由于损坏字符串永远不等于 `"日前电价"`，DA 分支被静默跳过，导致特征集不匹配、列名找不到。

### 同样问题

`utils/io.py` 也有编码损坏：`"鏃跺埢"` 应该是 `"时刻"`。

### 对比

`lightGBM/pipeline.py` 编码正确：`"dayahead": "日前电价"`, `"realtime": "实时电价"`。

`fusion/adapters/rt916.py` 已知此问题，做了兼容处理（同时接受正确和损坏两种值）。

### 修复方案

```python
# RT916_SpikeFusionNet/pipeline.py
TARGET_MAP = {
    "dayahead": "日前电价",
    "realtime": "实时电价",
}

# utils/io.py
# 将 "鏃跺埢" 替换为 "时刻"
```
